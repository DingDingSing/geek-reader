#### 大O复杂度表示法

```T(n) = O(f(n))```
- T(n) 代码的执行时间
- f(n) 每行代码的执行事件总和。因为是一个公司 所以用 f(n) 来表示
- O 表示代码执行时间 T(n) 和 f(n) 成正比

```js
function sum(n: number): number {
  let sum = 0;
  let i = 1;
  for (; i <= n; i++) {
    sum = sum + i;
  }
}
```
- 分析
  - ```T(n) = O(2n+2)```

```js
function cal(n: number) {
  let sum = 0;
  let i = 1;
  for (; i <= n; i++) {
    let j = 1;
    for (; j <= n; j++) {
      sum = sum + i * j;
    }
  }
}
```
- 分析
  - ``` T(n) = O(2n²+2n+2)```

#### 大O复杂度

> 代码执行时间随着数据规模增长的变化趋势 也叫渐进时间复杂度 简称时间复杂度

时间复杂度分析

- 只关注循环执行次数最多的一段代码
- 加法法则：总复杂度等于量级最大的那段代码的复杂度
- 乘法法则：嵌套代码复杂度等于嵌套内外代码复杂度的乘积
